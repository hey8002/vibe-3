# 공공직군 행정업무 슈퍼앱 PRD

## 1. 문서 목적
공공직군 행정업무 슈퍼앱의 초기 제품 요구사항을 정의한다. 본 문서는 구현 전에 기능 범위, 사용자, 핵심 시나리오, 우선순위, 수용 기준을 정리하기 위한 기준 문서이다.

## 2. 제품 개요
공공기관 또는 공공직군 조직에서 반복적으로 발생하는 일정 공유, 엑셀 정리, 민원 응대, 행정 뉴스 수집 업무를 하나의 웹 애플리케이션에서 처리하는 내부 업무용 슈퍼앱을 구축한다.

## 3. 목표
- 팀 단위 행정 일정 공유를 표준화한다.
- 반복 엑셀 작업을 자동화해 수작업 시간을 줄인다.
- 민원 매뉴얼 기반 응대 초안을 빠르게 생성한다.
- 공공 행정 관련 뉴스를 정기적으로 수집해 업무 참고 자료로 제공한다.

## 4. 주요 사용자
- 팀원: 본인 일정 확인, 공유 일정 확인, 엑셀 자동화 기능 사용
- 팀 관리자: 팀 전체 일정 관리, 공지성 일정 등록, 업무 현황 확인
- 민원 담당자: 민원 매뉴얼 기반 응대 스크립트 생성
- 행정 담당자: 매일 아침 수집된 뉴스 확인

## 5. 핵심 기능

### 5.1 팀원 스케줄 관리
팀원들의 휴가, 근무, 출장, 외근, 교육 등 일정을 공유하는 캘린더 기능을 제공한다.

#### 요구사항
- 월간, 주간, 일간 캘린더 조회
- 일정 유형 분류: 휴가, 근무, 출장, 외근, 교육, 기타
- 팀원별 일정 필터링
- 일정 등록, 수정, 삭제
- 일정 제목, 기간, 장소, 설명, 담당자 입력
- 중복 일정 또는 기간 겹침 표시

#### 수용 기준
- 사용자는 팀 전체 일정을 한 화면에서 확인할 수 있다.
- 사용자는 특정 팀원 또는 일정 유형만 필터링할 수 있다.
- 등록된 일정은 새로고침 후에도 유지되어야 한다.

### 5.2 엑셀 업무 자동화
엑셀 파일을 특정 컬럼 기준으로 나누거나 여러 엑셀 파일을 하나로 합치는 기능을 제공한다.

#### 요구사항
- `.xlsx` 파일 업로드
- 특정 컬럼 값 기준으로 파일 분리
- 여러 엑셀 파일 병합
- 병합 시 헤더 일치 여부 검증
- 작업 결과 파일 다운로드
- 오류 행 또는 누락 컬럼 안내

#### 수용 기준
- 사용자는 기준 컬럼을 선택해 엑셀 파일을 여러 파일로 분리할 수 있다.
- 사용자는 여러 엑셀 파일을 하나의 파일로 병합할 수 있다.
- 처리 실패 시 실패 사유를 화면에서 확인할 수 있다.

### 5.3 민원 업무 챗봇
첨부된 민원 매뉴얼을 바탕으로 민원 대응 방식, 답변 초안, 전화 응대 스크립트를 생성하는 챗봇 기능을 제공한다.

#### 요구사항
- 민원 매뉴얼 파일 업로드 또는 등록
- 사용자 질문 입력
- 매뉴얼 근거 기반 답변 생성
- 응대 스크립트 형식 제공
- 답변에 참고 근거 또는 출처 표시
- 민감정보 입력 주의 안내

#### 수용 기준
- 챗봇은 등록된 매뉴얼 범위 안에서 답변해야 한다.
- 답변에는 담당자가 검토해야 한다는 안내가 포함되어야 한다.
- 매뉴얼에 없는 내용은 임의로 단정하지 않고 확인 필요로 안내해야 한다.

### 5.4 뉴스 기사 수집
공공 행정과 관련된 뉴스들을 매일 아침 자동 수집해 제공한다.

#### 요구사항
- 수집 키워드 관리
- 매일 아침 정해진 시간에 뉴스 수집
- 제목, 언론사, 발행일, 링크, 요약 저장
- 중복 기사 제거
- 최신순 목록 조회
- 키워드별 필터링

#### 수용 기준
- 사용자는 당일 수집된 뉴스를 목록으로 확인할 수 있다.
- 동일 URL 또는 동일 제목 기사는 중복 저장되지 않아야 한다.
- 수집 실패 시 운영 문서 기준에 따라 원인을 확인할 수 있어야 한다.

## 6. MVP 범위
- 팀원 스케줄 CRUD 및 캘린더 조회
- 엑셀 분리 및 병합
- 민원 매뉴얼 기반 단순 질의응답
- 수동 실행 가능한 뉴스 수집 기능
- SQLite 기반 데이터 저장

## 7. 제외 범위
- 외부 그룹웨어 연동
- 전자결재 기능
- 사용자별 복잡한 권한 체계
- 모바일 앱 네이티브 개발
- 실시간 메신저 기능

## 8. 비기능 요구사항
- 브라우저 기반 웹 애플리케이션으로 제공한다.
- 프론트엔드는 TypeScript, Vite, React를 사용한다.
- 백엔드는 Python, FastAPI, uv를 사용한다.
- 데이터베이스는 SQLite를 사용한다.
- 파일 업로드 시 허용 확장자와 크기 제한을 적용한다.
- 민원 데이터와 개인정보는 로그에 남기지 않는 것을 원칙으로 한다.

## 9. 우선순위
1. 프로젝트 기본 구조와 실행 환경 구성
2. 팀원 스케줄 관리
3. 엑셀 업무 자동화
4. 뉴스 기사 수집
5. 민원 업무 챗봇

## 10. 주요 리스크
- 민원 매뉴얼 기반 답변의 정확성 검증 필요
- 뉴스 수집 대상 사이트의 구조 변경 가능성
- 엑셀 파일 형식 다양성으로 인한 예외 처리 증가
- SQLite 동시 쓰기 제한에 따른 운영 범위 관리 필요
