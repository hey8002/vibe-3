# FE 테스트 작성

## 목적

React + TypeScript + Vite 프런트엔드의 동작을 검증하는 테스트를 작성하고 실행한다.

## 사용 원칙

- 사용자가 명시적으로 테스트 작성 또는 실행을 요청한 경우에만 사용한다.
- 프로젝트에 이미 있는 테스트 도구와 설정을 우선 따른다.
- 기본 우선순위는 `vitest`, 다음은 `jest`, 그 외에는 레포에 이미 존재하는 도구다.
- 테스트를 위해 제품 코드를 과도하게 바꾸지 않는다.
- 통과 여부뿐 아니라 실패 원인도 함께 기록한다.

## 레포 확인 기준

- `frontend/package.json`에 테스트 스크립트가 있으면 그 스크립트를 우선 실행한다.
- `vitest` 설정 파일이 있으면 `vitest` 기준으로 작성한다.
- `jest` 설정 파일이 있으면 `jest` 기준으로 작성한다.
- 테스트 도구가 없으면 새 도구를 임의로 도입하지 말고, 현재 상태와 필요한 추가 작업을 구분해서 보고한다.

## 작업 절차

1. 관련 컴포넌트, 훅, 유틸을 확인한다.
2. 기존 테스트와 테스트 설정을 확인한다.
3. 필요한 테스트를 추가하거나 수정한다.
4. 관련 테스트만 우선 실행한다.
5. 실패가 있으면 원인을 분석하고 최소 수정 방향을 제안한다.
6. 최종 결과를 표로 정리한다.

## 우선 점검 항목

- 렌더링 결과
- 사용자 이벤트
- 상태 변화
- props 분기
- 경계값
- 예외 처리

## 결과 형식

항상 표로 정리한다.

| 구분 | 파일 | 내용 | 결과 | 제안 |
| --- | --- | --- | --- | --- |
| 테스트 추가 | 경로 | 무엇을 검증했는지 | 통과 / 실패 | 필요한 수정 |
| 테스트 실패 | 경로 | 실패 원인 | 실패 | 수정 방향 |

## 주의사항

- 실행하지 않은 테스트는 실행한 것처럼 말하지 않는다.
- 스냅샷은 꼭 필요한 경우에만 사용한다.
- 불안정한 테스트는 원인을 함께 표시한다.
