# BE 테스트 작성

## 목적

uv 기반 Python+FastAPI 백엔드의 동작을 검증하는 테스트를 작성하고 실행한다.

## 사용 원칙

- 사용자가 명시적으로 테스트 작성 또는 실행을 요청한 경우에만 사용한다.
- 프로젝트에 이미 있는 테스트 도구와 설정을 우선 따른다.
- 기본 우선순위는 `pytest`다.
- 테스트를 위해 제품 코드를 과도하게 바꾸지 않는다.
- 통과 여부뿐 아니라 실패 원인도 함께 기록한다.

## 레포 확인 기준

- `backend/pyproject.toml` 또는 관련 설정에 테스트 도구가 있으면 그 도구를 우선한다.
- `pytest` 설정 파일이 있으면 `pytest` 기준으로 작성한다.
- 테스트 도구가 없으면 새 도구를 임의로 도입하지 말고, 현재 상태와 필요한 추가 작업을 구분해서 보고한다.
- `uv` 기반 실행 환경이면 `uv run` 또는 레포에 존재하는 실행 방식을 우선 확인한다.

## 작업 절차

1. 관련 라우트, 서비스, 의존성, 유효성 검증 로직을 확인한다.
2. 기존 테스트와 테스트 설정을 확인한다.
3. 필요한 테스트를 추가하거나 수정한다.
4. 관련 테스트만 우선 실행한다.
5. 실패가 있으면 원인을 분석하고 최소 수정 방향을 제안한다.
6. 최종 결과를 표로 정리한다.

## 우선 점검 항목

- 라우트 동작
- 서비스 로직
- 의존성 주입
- 입력 검증
- 예외 처리
- 경계값

## 결과 형식

항상 표로 정리한다.

| 구분 | 파일 | 내용 | 결과 | 제안 |
| --- | --- | --- | --- | --- |
| 테스트 추가 | 경로 | 무엇을 검증했는지 | 통과 / 실패 | 필요한 수정 |
| 테스트 실패 | 경로 | 실패 원인 | 실패 | 수정 방향 |

## 주의사항

- 실행하지 않은 테스트는 실행한 것처럼 말하지 않는다.
- 불안정한 테스트는 원인을 함께 표시한다.
- 보안을 위한 입력 검증 테스트는 우선적으로 확인한다.
