# 코드 리뷰

## 목적

코드 변경이 끝난 뒤, FE와 BE 코드를 읽고 품질 이슈를 식별한다.

## 사용 원칙

- 사용자가 `코드 리뷰` 또는 동등한 의미로 명시적으로 요청한 경우에만 이 스킬을 사용한다.
- FE는 `React + TypeScript + Vite`, BE는 `uv` 기반 Python+FastAPI로 본다.
- 추측보다 코드 근거를 우선한다.
- 변경 의도를 훼손하지 않는 범위에서만 문제를 지적한다.

## 리뷰 항목

1. 직관성
2. 보안 취약성
3. 재사용성

## 리뷰 절차

1. 변경 범위와 관련 파일을 확인한다.
2. FE와 BE를 분리해서 읽는다.
3. 각 항목에 대해 문제 여부를 판단한다.
4. 문제는 영향도와 함께 간단히 설명한다.
5. 가능한 경우 구체적인 수정 방향을 제안한다.

## 결과 형식

항상 표로 정리한다.

| 구분 | 파일 | 발견 내용 | 영향도 | 제안 |
| --- | --- | --- | --- | --- |
| FE / BE | 경로 | 핵심 이슈 | 높음 / 중간 / 낮음 | 수정 방향 |

## 보고 기준

- 이슈가 없으면 표에 `없음`을 적는다.
- 자잘한 스타일 취향은 제외하고 실제 품질 문제만 적는다.
- 보안 이슈는 우선적으로 표시한다.
